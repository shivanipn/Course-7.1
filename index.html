“DBMS And MySql ”

Introduction

A database is a system for storing and managing data. It is widely used in businesses and industries, such as banking, supermarkets, and ticket booking, to store customer information. A Database Management System (DBMS) helps in organizing and handling this data efficiently.

Database

A database is an organized collection of data that allows for easy storage, access, processing, and analysis. It structures data into tables with rows and columns, making management efficient. Databases are widely used in organizations for applications like library systems and flight reservations.

Database Management System (DBMS)

Database Management System or DBMS in short refers to the technology of storing and retrieving users' data with utmost efficiency along with appropriate security measures. This tutorial explains the basics of DBMS such as its architecture, data models, data schema, data independence, E-R model, relation model, relational database design, and storage and file structure and much more.

Need for DBMS

  • Data retrieval:If you want to retrieve data from the flat file then we must develop application programs in a high-level language, so that data can be stored and retrieved fastly and securely within the time bound Ex: SQL – structured query language
  • Data redundancy : In any storage, we need to make copies of data for backup but in traditional file management systems once we update data in one location sometimes it fails to get updated in the copy of the data, so that it may create problems of inconsistency this rate is called duplicate data or redundant data
  • Data integrity:Data integrity ensure that only required data is stored in the database.e data is validated before entered into the database using integrity constraints such as primary key, foreign key, etc.
  • Data security: In traditional file management, there is no authentication mechanism at high-end whereas DBMS provides levels of security authentication which can be done at user level admin level, etc
  • Data indexing: If you want to retrieve data very fastly from the database we are using indexing mechanism whereas Flat files don’t support indexing and solely depend upon secondary storage devices
Relational DBMS Terminology

Before proceeding to MySQL database system, let us understand the few terms related to the database.
Database - A database is a collection of tables, with related data.
Table - A table is a combination of rows and columns with collection of related data entries. A table in a database looks like a simple spread sheet.
Row -Row - A row, also called as tuple, entry or record represents a single data item in a table and it is a set of related data. For example, if a table represents schools then each row would represent single school.
Field - Each row of a table consists of several smaller entities which are known as fields. It can also be called single item of a column. Therefore, a field is a joint of row and column.
Column -A column, also called as an attribute or field is a set of values of particular type for each row of a table. Each column holds data value of particular type. For example, address or contact number.
Redundancy - Redundancy is the appearance of the same data factor in more than one field or table of data. The redundancy of data produces inconsistency. Therefore, normalisation is done to eliminate the redundancy.

  • Primary Key:A primary key is a column or set of columns in a table that uniquely identifies rows in a table. A primary key is unique and key value cannot occur twice in one table.
  • Foreign Key: A foreign key is a column or set of columns in a table that points to the primary key of another table. A foreign key is the linking pin between two tables. Unlike primary keys, foreign keys can contain duplicate values. A table can have more than one foreign key. .
  • Candidate Key:Candidate key is a set of columns from which primary key can be selected. It is a set of columns of which each column can act as a primary key for a table to uniquely identify each row in a table.
  • Composite Key: A composite key (compound key) is a key that consists of two or more columns to uniquely identify rows in a table.
Database Language

Database languages are special languages used to create and manage databases on computers. These languages allow us to do operations on the data stored in the database. Different database languages are available for various data models.

    The different types of DBMS languages are as follows −
  • Data Definition Language (DDL):Create, Drop, Truncate, Rename.
  • Data Manipulation language (DML): Select, Insert, Delete, Update.
  • Data Control Language (DCL): Revoke, Grant.
  • Transaction Control Language (TCL):Rollback, Commit
Entity Relationalship Diagram

An Entity Relationship (ER) Diagram is a type of flowchart that illustrates how “entities” such as people, objects or concepts relate to each other within a system. ER Diagrams are most often used to design or debug relational databases in the fields of software engineering, business information systems, education and research. Also known as ERDs or ER Models, they use a defined set of symbols such as rectangles, diamonds, ovals and connecting lines to depict the interconnectedness of entities, relationships and their attributes. They mirror grammatical structure, with entities as nouns and relationships as verbs.

MySQL Databse

MySQL is an open source RDBMS that uses SQL to create and manage databases. As a relational database, MySQL stores data in tables of rows and columns organized into schemas. A schema defines how data is organized and stored and describes the relationship among various tables. With this format, developers can easily store, retrieve, and analyze many data types, including simple text, numbers, dates, times, and, more recently, JSON and vectors.

SQL Data types

An SQL data type refers to the type of data which can be stored in a column of a database table. In a column, the user can store numeric, string, binary, etc by defining data types. For example integer data, character data, monetary data, date and time data, binary strings, and so on.
While creating a database table in a database, we need to specify following two attributes to define a table column:

  • Name of the column
  • Data type of the column

Types of SQL Data Types:

  • String
  • Numeric
  • Date and Time
SQL Statements

SQL statements, or SQL queries, are valid instructions that relational database management systems understand. Software developers build SQL statements by using different SQL language elements. SQL language elements are components such as identifiers, variables, and search conditions that form a correct SQL statement.
For example, the following SQL statement uses a SQL INSERT command to store Mattress Brand A, priced $499, into a table named Mattress_table, with column names brand_name and cost:
INSERT INTO Mattress_table (brand_name, cost)
VALUES(‘A’,’499’);

Some of The Most Important SQL Commands
  • SELECT - extracts data from a database
  • UPDATE - updates data in a database
  • DELETE - deletes data from a database
  • INSERT INTO - inserts new data into a database
  • CREATE DATABASE - creates a new database
  • ALTER DATABASE - modifies a database
  • CREATE TABLE - creates a new table
  • ALTER TABLE - modifies a table
  • DROP TABLE - deletes a table
  • CREATE INDEX - creates an index (search key)
  • DROP INDEX - deletes an index
